# Training Pipeline

CueZero uses a three-stage training pipeline to iteratively improve its performance. The system combines supervised pre-training with self-play reinforcement learning.

## Training Overview

| Phase | Description | Duration |
|-------|-------------|----------|
| **Pre-training** | Supervised learning on BasicAgent data | ~200 epochs |
| **Self-play Training** | AI vs AI with MCTS-guided data generation | ~600 epochs |
| **Supplementary Training** | Specialized refinement and data polishing | ~200 epochs |
| **Total** | | **~1000 epochs** |

## Key Training Statistics

- **Basic proficiency achieved**: ~200 epochs (learns fundamental shot-making)
- **Total training iterations**: ~1000 epochs
- **Self-play games per iteration**: 400 games
- **Training data efficiency**: Heuristic search accelerates learning by ~3-5x

---

## Phase 1: Pre-training

### Goal
Initialize the policy and value networks with supervised learning to establish basic shot-making capabilities.

### Data Source
- **10,000 games** generated by BasicAgent self-play
- Each game provides state-action-outcome tuples

### Network Architecture
```
Input: 3 consecutive states (3 × 81 dimensions)
  ↓
Shared Convolutional Feature Extractor
  ├─→ Policy Head → 5-dimensional action vector
  └─→ Value Head → Win probability [0, 1]
```

### Training Configuration
```yaml
# configs/training.yaml
training:
  learning_rate: 0.001
  batch_size: 64
  epochs: 200
  optimizer: Adam

data:
  buffer_size: 100000
```

### Command
```bash
# Pre-training on BasicAgent data
python scripts/train.py --pretrain --data-path data/basic_agent_games/
```

## Phase 2: Self-play Training

### Goal
Achieve high-level performance through iterative self-improvement using MCTS-guided data generation.

### Closed-loop Training Pipeline

```
┌─────────────────┐    Generate    ┌─────────────────┐
│  Current Model  │ ──────────────→ │  Self-play Games │
└─────────────────┘                 └────────┬────────┘
         ↑                                   │
         │ Train                             │
         │                                   │
┌────────┴────────┐   Improve    ┌─────────▼─────────┐
│  Updated Model   │ ←──────────  │  Training Dataset  │
└─────────────────┘              └─────────────────────┘
```

### Key Innovations

#### 1. Heuristic Action Generation
- **Ghost Ball method**: Geometrically calculates optimal shot angles
- **Candidate filtering**: Policy network prunes low-value actions (~66% reduction)
- **Result**: Faster convergence by focusing on promising actions

#### 2. Hybrid Value Estimation
- Combines neural network predictions with physical simulation outcomes
- Dynamic weighting: Early training → more simulation; Late training → more network
- Reduces variance in value estimates

#### 3. Dynamic Search Strategy
- As model improves, rely more on policy outputs for candidate pruning
- Use more accurate value predictions to reduce multi-step simulation error

### Self-play Configuration
```yaml
# configs/mcts.yaml
search:
  num_simulations: 150      # Full mode
  exploration_weight: 0.25
  temperature: 1.0

action:
  num_action_samples: 30     # Heuristic candidates
  max_depth: 4
```

### Command
```bash
# Generate self-play data
python scripts/selfplay.py --num-games 400 --output-dir data/selfplay/

# Train on self-play data
python scripts/train.py --selfplay --data-path data/selfplay/
```

## Phase 3: Supplementary Training

### Goal
Polish performance with specialized training on edge cases and high-value scenarios.

### Specialized Training Strategies

#### 1. Special Situation Training
Focus on challenging scenarios:
- Remaining target balls ≤ 3
- Cue ball against the cushion
- Only 8-ball remaining

#### 2. Data Quality Filtering
Remove low-quality samples:
- Immediate fouls on break
- Consecutive empty hits
- Irrecoverable mistakes

#### 3. Policy Distillation
- Re-generate optimal actions for mistake states using expanded MCTS
- Replace erroneous actions in training dataset
- Boosts performance by ~5-10%

### Command
```bash
# Supplementary training with specialized data
python scripts/train.py --supplementary --data-path data/specialized/
```

```bash

tail -f logs/training.log

ls -lh models/checkpoints/
```

### Configuration Files

All training parameters are configurable via YAML files:

| File | Purpose |
|------|---------|
| `configs/model.yaml` | Network architecture dimensions |
| `configs/training.yaml` | Learning rate, batch size, epochs |
| `configs/mcts.yaml` | MCTS search parameters |

### Hardware Recommendations

- **Pre-training**: Consumer GPU (NVIDIA RTX 30xx/40xx)
- **Self-play**: Multi-core CPU for parallel simulation
- **Full training**: Siyuan-1 cluster (Xeon 8358 + A100)
