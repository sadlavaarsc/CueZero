CueZero is a reinforcement learning system for billiards that combines neural networks with a continuous-action Monte Carlo Tree Search (MCTS) to solve a high-dimensional decision-making problem.

## Project Overview

CueZero is a high-performance billiards AI designed for continuous control tasks. Unlike traditional AlphaZero implementations for discrete games, CueZero handles a 5-dimensional continuous action space and an 81-dimensional state space.

The system integrates:

- Policy-value neural networks
- Continuous-action MCTS
- Physics-based simulation

To achieve strong performance in a stochastic and highly dynamic environment.

## Key Features

- **Continuous-action MCTS**: Extends traditional MCTS to a 5D continuous action space with policy-guided local search
- **Policy-value network**: Learns both action distribution and state value from 81D state inputs
- **Self-play pipeline**: Automated data generation and iterative model improvement
- **Physics simulation integration**: Uses pooltool for accurate environment modeling
- **Hybrid evaluation**: Combines learned value estimation with simulation-based rollouts

## Architecture

![Architecture](./assets/architecture.png)

The overall system follows a neural-guided search pipeline:

CueZero's architecture consists of four core components:

1. **Policy Network**: Takes 81-dimensional state vectors (containing ball positions, velocities, and game state) as input and outputs 5-dimensional action vectors (velocity, angle, etc.)
2. **Value Network**: Shares the same convolutional feature extractor as the policy network and predicts the probability of winning from the current state
3. **Monte Carlo Tree Search**: Optimized for continuous action spaces, using the policy network to guide search and the value network to evaluate positions
4. **Closed-loop training**: Combines pre-training on expert data with self-play to continuously improve model performance

## Training Pipeline

![training pipeline](./assets/training_pipeline.png)

1. **Prior training**: Initializes networks with supervised learning on data generated by BasicAgent
2. **Self-play training**: AI plays against itself to generate high-quality training data
3. **Dataset building**: Processes and prepares match data for training
4. **Model training**: Trains neural networks on the generated data
5. **Evaluation**: Assesses model performance against baseline agents

## Core Challenges and Solutions

| Challenge                                  | Solution                                                                 |
| ------------------------------------------ | ------------------------------------------------------------------------ |
| High dimensionality (81D state, 5D action) | Custom neural network architecture with convolutional feature extraction |
| Continuous action space                    | Policy-guided MCTS with local neighborhood search                        |
| Strong uncertainty                         | Hybrid evaluation combining neural predictions and physical simulation   |
| Real-time requirements                     | Optimized MCTS with pruning and limited node expansion                   |
| Chaotic behavior                           | Ghost ball heuristic + random perturbation sampling + policy filtering   |

## Quick Start

### Environment Setup

```bash
# Create conda environment (Python 3.10+ recommended)
# Note: This project uses the 'poolenv' environment
conda activate poolenv

# If you don't have the environment yet, create it:
conda create -n poolenv python=3.10
conda activate poolenv

# Install pooltool (billiards physics engine)
# Install from source (recommended):
git clone https://github.com/bboland/pooltool.git
cd pooltool && pip install -e .
cd ..

# Install dependencies
pip install -r requirements.txt

# Install the package (optional, for development)
pip install -e .
```

### Web UI

CueZero includes a web-based UI for visualizing and interacting with the AI.

```bash
# Start the web server (mock mode - default)
# Run from project root, no need to cd into server directory
PYTHONPATH=. python -m server.server

# Start with AI mode (requires dual_network_final.pt model file)
PYTHONPATH=. python -m server.server --ai

# Access the UI at http://localhost:8000
```

The web UI supports:
- Visualizing the billiard table and ball positions
- Executing shots and seeing results
- Getting AI-recommended shots (in AI mode)

### CLI Battle

CueZero provides a command-line interface for agent battles.

```bash
# Quick start: MCTS(Fast) vs Basic
python scripts/cli_game.py --agent-a mcts_fast --agent-b basic --games 5

# Human vs MCTS(Full)
python scripts/cli_game.py --agent-a human --agent-b mcts_full --games 3

# View all options
python scripts/cli_game.py --help
```

**Arguments**:
- `--agent-a`: Agent A type (human, mcts_fast, mcts_full, policy, basic, random)
- `--agent-b`: Agent B type
- `--games`: Number of games to play
- `--model`: Path to model file
- `--no-verbose`: Disable verbose output
- `--seed`: Random seed

### Agent Types

| Type       | Description                                   | Use Case                  |
|------------|-----------------------------------------------|---------------------------|
| human      | Human player via command line input           | Human vs AI, testing      |
| mcts_fast  | MCTS fast mode (30 simulations, depth 2)      | Real-time play, quick test|
| mcts_full  | MCTS full mode (150 simulations, depth 4)     | Strong play, offline analysis |
| policy     | Policy network direct prediction              | Fast inference            |
| basic      | Heuristic rule-based                          | Baseline comparison       |
| random     | Random actions                                | Testing, debugging        |

### Server Battle API

The server supports agent battles via REST API.

```bash
# Start server with default agents
python server.py --agent-a mcts_fast --agent-b basic

# Start a battle via API
curl -X POST http://localhost:8000/api/battle/start \
  -H "Content-Type: application/json" \
  -d '{"agent_a_type": "human", "agent_b_type": "mcts_fast", "total_games": 3}'

# Execute next step (auto AI decision)
curl -X POST http://localhost:8000/api/battle/{battle_id}/next \
  -H "Content-Type: application/json" \
  -d '{}'

# Get battle status
curl http://localhost:8000/api/battle/{battle_id}/status

# List all battles
curl http://localhost:8000/api/battles
```

### MCTS Modes

The MCTS implementation supports two modes for different use cases:

```python
from cuezero.mcts.search import MCTS

# Fast mode: Quick decisions (30 simulations, depth 2, 3s timeout)
mcts_fast = MCTS(model=model, mode="fast")

# Full mode: Strong play (150 simulations, depth 4, 15s timeout)
mcts_full = MCTS(model=model, mode="full")

# Custom configuration (overrides mode defaults)
mcts_custom = MCTS(model=model, mode="fast", n_simulations=50, max_depth=3)
```

| Mode | Simulations | Max Depth | Timeout | Use Case |
|------|-------------|-----------|---------|----------|
| fast | 30 | 2 | 3s | Real-time play, web UI |
| full | 150 | 4 | 15s | Offline analysis, strong play |

## Model Files

The pre-trained model file `dual_network_final.pt` should be placed in the project root directory.

The model architecture consists of:
- **SharedFeatureExtractor**: Processes 3x81 dimensional state vectors (3 consecutive game states)
- **PolicyHead**: Outputs 5-dimensional action vector (velocity, angles, offsets)
- **ValueHead**: Outputs win probability estimate

Model file format: PyTorch state dict with `feature_extractor`, `policy_head`, and `value_head` components.

## Future Directions

### System & Engineering

- **Modular service architecture**: Refactor training, self-play, and inference into independent services with clear interfaces (e.g., RPC/REST), enabling scalable deployment
- **Distributed self-play**: Parallelize data generation across multiple workers or machines to improve training throughput
- **Experiment management**: Integrate tools for configuration tracking, logging, and reproducibility (e.g., structured configs, versioned checkpoints)
- **Performance optimization**: Optimize MCTS and simulation bottlenecks via batching, caching, and mixed-precision inference
- **Model serving & deployment**: Expose trained agents via inference APIs for real-time decision-making or external integration
- **Visualization & debugging tools**: Build tools for trajectory replay, decision inspection, and training diagnostics

### Algorithm & Modeling

- **Improved policy representation**: Explore more expressive models (e.g., attention mechanisms) for better spatial reasoning
- **Efficient continuous search**: Investigate more sample-efficient exploration strategies in continuous action spaces
- **Uncertainty modeling**: Incorporate stochastic modeling or confidence estimation into value prediction
- **Curriculum learning**: Design progressive training schemes to stabilize learning in complex environments
- **Hybrid methods**: Combine learning-based approaches with analytical or physics-based priors
